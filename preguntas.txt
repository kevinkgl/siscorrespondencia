

  El problema técnico:
  Los navegadores web (Chrome, Firefox, etc.) bloquean las conexiones directas a bases de datos SQL (como        
  PostgreSQL) por razones de seguridad (CORS y falta de soporte de sockets TCP directos). Tu código actual de    
  postgres: ^3.0.0 funcionará en Windows/Android, pero fallará en la Web al intentar conectar.

  ---

  Plan de Acción: Conectar Flutter Web a PostgreSQL

  Para que tu aplicación web use tu base de datos actual, necesitamos un intermediario (API). Aquí tienes el plan  paso a paso:


  1. Arquitectura de la Solución
  En lugar de App -> PostgreSQL, usaremos:
  `Flutter Web (HTTPS)` <---> `API REST (Node.js/Python/Dart)` <---> `PostgreSQL`

  2. Implementación Técnica (Fases)


  Fase A: Crear el Servidor API (El "Puente")
  Tienes dos opciones principales:
   1. Dart Frog o Shelf (Recomendado): Permite usar Dart en el servidor y reutilizar tus modelos de datos.       
   2. Node.js (Express): Muy común y fácil de desplegar en Firebase Functions o Render.


  Fase B: Crear Endpoints de Autenticación
  Debes crear rutas en tu API como:
   * POST /login: Recibe usuario/clave, consulta a Postgres, verifica con bcrypt y devuelve un Token (JWT).      
   * GET /correspondencia: Recibe el Token y devuelve los datos de la DB.


  Fase C: Adaptar Flutter (Frontend)
  Debes modificar tu lógica de conexión para detectar si estás en la Web:
   1 if (kIsWeb) {
   2   // Usar http.post('https://tu-api.com/login')
   3 } else {
   4   // Seguir usando tu conexión directa a Postgres (Desktop/Mobile)  
   5 }

  Plan de Acción para un Sistema Multisede y Remoto


  1. Centralización (La Nube)
  Para que cualquier máquina en el país se conecte, la base de datos y la API deben estar en un servidor 24/7 con  IP pública.
   * Base de Datos: Migrar tu Postgres local a Supabase o Neon.tech (Postgres gratuito/económico en la nube).    
   * API (Puente): Desplegar la carpeta backend que creamos a Firebase Functions o Render.


  2. Adaptación para "Sucursales" (Lógica de Negocio)
  Para manejar varias sucursales en una sola base de datos, debes aplicar un modelo Multi-tenant:
   * Base de Datos: Cada tabla importante (Usuarios, Correspondencia, Inventario) debe tener una columna       
     id_sucursal.
   * Login: Cuando un empleado inicia sesión, la API debe devolver a qué sucursal pertenece.
   * Filtros: Todas las consultas SQL deben incluir un WHERE id_sucursal = X para que una sede no vea los datos
     de otra.


  3. Instalación Universal
   * Web: Ya está lista en Firebase Hosting. Cualquier navegador puede entrar.
   * Desktop (Windows): Puedes generar un instalador (.exe) que se conecte a la misma API.
   * Mobile (Android): El mismo código funcionará para los repartidores o gerentes.

   6

   9 ALTER TABLE correspondencia ADD COLUMN id_sucursal INTEGER REFERENCES sucursales(id);


  Paso 2: Unificar Flutter con el ApiClient
  Debes dejar de usar DatabaseService directamente en tus pantallas. Todo debe pasar por ApiClient.
  Ejemplo de cambio:
   * Antes: db.connection.execute("SELECT * FROM...")
   * Ahora: apiClient.query("SELECT * FROM...")


  Paso 3: Desplegar la API (El Puente) a Internet
  Para que no dependas de tu PC, vamos a subir el backend.
   1. Crear una cuenta en Render.com (https://render.com) (es gratis para APIs de Node.js).
   2. Conectar tu repositorio de GitHub.
   3. Tu API tendrá una URL como: https://sistema-api.onrender.com.
   4. Actualizar la baseUrl en tu lib/core/api/api_client.dart.


  Paso 4: Seguridad (HTTPS / JWT)
  Al estar en internet, cualquiera podría intentar atacar tu base de datos.
   * La API que creamos ya usa JWT (Tokens). Debes asegurarte de que cada consulta a la API pida el token que se
     generó al hacer login.
